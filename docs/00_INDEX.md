# Dokumentindex

Detta är den centrala dokumentstrukturen för projektets krav, planering och ändringshistorik.

## Läsordning
1. `docs/01_MASTER_SPEC.md` – vad som ska byggas och hur det ska fungera.
2. `docs/02_BACKLOG.md` – prioriterade arbetsuppgifter.
3. `docs/03_BESLUT_LOGG.md` – beslut och motivering.
4. `docs/04_CHANGELOG.md` – vad som faktiskt har ändrats.

## Arbetsflöde
1. Uppdatera spec vid nya krav.
2. Lägg eller uppdatera uppgifter i backlog.
3. Implementera i kod.
4. Verifiera.
5. Dokumentera i changelog.
